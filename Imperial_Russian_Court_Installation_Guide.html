Imperial Russian Court System - Installation Guide

This comprehensive guide will walk you through the process of setting up the Imperial Russian Court roleplay system in Second Life, step by step. Follow these instructions carefully to ensure proper installation and functionality.

Note: Before beginning installation, make sure you have building permissions in your Second Life region and basic knowledge of LSL (Linden Scripting Language) concepts.

1. System Overview

The Imperial Russian Court system consists of multiple interconnected components:

  • Core HUD: The central script coordinating all functionality
  • Module Scripts: Specialized scripts for different aspects of gameplay
  • Special HUDs: Custom HUDs for characters like the Tsar, Tsarina, Grand Duchesses, etc.
  • Public Meter: Displays character information to other players
  • NPC Characters: Interactive non-player characters with specific behaviors
  • Accessories: Additional items like weapons, food, sacred relics, etc.
Important: The HUD and Core System are actually the same physical object. The HUD is the physical object you attach to your avatar, while the Core System refers to the main controlling script within that object.

2. Creating the HUD Object

1 Create a new prim

In Second Life, right-click on the ground and select "Create" to make a new prim. A simple box shape works well as a starting point for your HUD.

2 Size and position the prim

Adjust the prim to an appropriate size for a HUD. Typical dimensions might be:

  • X: 0.3 meters
  • Y: 0.3 meters
  • Z: 0.05 meters

This creates a small, flat object that won't be too intrusive when attached.

3 Add additional prims for modules

Create additional prims and link them to the main prim to create a multi-part HUD. This allows for a more organized structure, with each module potentially having its own prim.

Create at least one additional prim for each major module you plan to include:

  • One prim for Family Module
  • One prim for Religion Module
  • One prim for Banking Module
  • One prim for Politics Module
  • Additional prims as needed for other modules
4 Link the prims

Select all the prims you've created, with the main HUD prim selected last (it will have a yellow highlight instead of blue). Then press Ctrl+L to link them together into a single object.

The last-selected prim becomes the "root prim" that will contain the Core System script.

5 Set HUD properties

Right-click the linked object and select "Edit". In the "General" tab, check the following settings:

  • Check "Phantom" to ensure the HUD doesn't interact physically with the environment
  • Make sure "Temporary" is unchecked
  • Set the object name to "Imperial Russian Court HUD"
Important: Do not set the object to "Physical" as this can cause issues when attached as a HUD.

3. Installing the Core System

1 Open the contents of the root prim

Right-click on your linked HUD object and select "Edit". Then click the "Contents" tab to view and edit the contents of the object.

2 Create the Core HUD script

Click the "New Script" button in the contents tab. This creates a new script inside the object.

Rename the script to "Imperial_Russian_Court_Core_HUD" by right-clicking on it and selecting "Rename".

Double-click the script to open it for editing.

3 Replace script content

Delete all the default script content and replace it with the Imperial_Russian_Court_Core_HUD.lsl script content that we've developed.

Save the script with Ctrl+S or by clicking the Save button.

4 Add the System Connector script

Create another new script in the same root prim.

Rename it to "Imperial_System_Connector_Fixed".

Replace its content with our Imperial_System_Connector_Fixed.lsl script.

Save the script.

Note: The System Connector script handles communication between all modules and prevents memory issues. It must be installed in the same prim as the Core HUD script.
5 Set up the HUD attachment point

To enable the object to be worn as a HUD:

  • Right-click the linked object and select "Attach to HUD" → Select your preferred position (typically "Top Right")
  • Alternatively, take the object into your inventory, right-click it there, and select "Attach to HUD" → Choose position

You can adjust the precise position once attached by editing it while worn.

4. Installing Additional Modules

Each module should be installed in its own prim within the linked HUD object. This helps manage memory usage and prevents script size issues.

1 Access each module prim

Edit the linked HUD object and select each child prim individually to add the appropriate module scripts.

You can select individual prims in a linked object by:

  • Right-clicking the object and selecting "Edit"
  • In Edit mode, click on the specific prim you want to modify
  • Click the "Contents" tab to access that specific prim's contents
2 Add module scripts

For each module, add a new script to the appropriate prim, rename it to match the module name, and paste in the corresponding script content.

6. Setting Up Foreign Visitors Module

The Foreign Visitors Module allows players from other courts and countries to register with the Imperial Russian Court and receive appropriate diplomatic treatment and benefits.

1 Create the Foreign Visitors Registration Bureau

Create a new object to serve as the registration bureau for foreign visitors:

  • Create a prim with an appropriate design (desk, bureau, or registration book)
  • Name it "Imperial Foreign Visitors Registration Bureau"
  • Make it big enough to be easily visible in your roleplay area
2 Install the Foreign Visitors Module Script

Add a new script to the registration bureau:

  1. Create a new script in the contents of the object
  2. Rename it to "Imperial_Foreign_Visitors_Module"
  3. Replace its content with the Imperial_Foreign_Visitors_Module.lsl script
  4. Save the script
3 Create the Foreign Visitors Connector

Add another script to the registration bureau to connect it with the rest of the system:

  1. Create another new script in the contents of the object
  2. Rename it to "Imperial_Foreign_Visitors_Connector"
  3. Replace its content with the Imperial_Foreign_Visitors_Connector.lsl script
  4. Save the script
4 Create the Foreign Visitor HUD

Create a special HUD for foreign visitors to wear:

  1. Create a new HUD object following the same basic process as the standard HUD creation
  2. Label it "Imperial Foreign Visitor HUD"
  3. Add a new script to the HUD
  4. Rename it to "Imperial_Foreign_Visitor_HUD"
  5. Replace its content with the Imperial_Foreign_Visitor_HUD.lsl script
  6. Save the script
5 Position the registration bureau

Place the Imperial Foreign Visitors Registration Bureau in a prominent location in your court area, such as:

  • Near the entrance to the palace or court area
  • In a dedicated diplomatic reception room
  • Near the main throne room or audience chamber
Features: The Foreign Visitors Module allows visitors to register their name, country, court/government, rank/position, and title. Upon registration, they receive 3,000 rubles as a diplomatic stipend and their information is integrated with the court relationship system.
Diplomatic Relations: The Foreign Visitors Module includes pre-configured diplomatic relations between Russia and other major powers as of 1905, including strained relations with Germany and Japan, and strong alliance with France. These can be adjusted through the system as diplomatic events unfold in roleplay.
  • Hemophilia Management System: Tracks bleeding risk, triggers bleeding episodes from injuries or dangerous activities, and requires medical care
  • Activity System: Different activities have different risk levels for triggering bleeding episodes
  • Caregiver System: Allows certain players to be designated as caregivers who can help manage bleeding episodes
  • Daily Health Changes: Each day has a baseline risk level that affects vulnerability to injuries
  • Special Commands: Includes chat commands like /rest, /activity, /caregiver, and /stats for managing the condition
Historical Note: The real Tsarevich Aleksei suffered from hemophilia, a genetic blood disorder that prevented normal blood clotting. The condition was kept secret from the Russian public and greatly influenced the Romanov family dynamics, including the rise of Rasputin as a healer.

7. Installing the Public Meter

1 Create the meter object

Create a new object that will serve as the public meter. This is typically a flat, rectangular prim that will display text above the player's head.

Recommended dimensions:

  • X: 0.3 meters
  • Y: 0.02 meters
  • Z: 0.1 meters

Set the object to "Phantom" to prevent it from interacting physically with the environment.

2 Install the meter script

Create a new script in the meter object and rename it to "Imperial_Court_Meter_Updated".

Replace the script content with the Imperial_Court_Meter_Updated.lsl script.

Save the script.

3 Set up attachment point

Configure the meter to attach above the avatar's head:

  • Right-click the meter object and select "Attach" → "Avatar Center" or "Head"
  • Adjust the position while worn to make sure it displays properly above the head
4 Link meter with HUD

The meter communicates with the HUD via chat channels. Make sure the channels defined in both scripts match:

  • Check METER_CHANNEL variable in both the Core HUD script and Meter script
  • They should be set to the same negative integer value (e.g., -8675309)

10. Testing and Troubleshooting

1 Basic functionality test

After installing all components, perform a basic test:

  1. Attach the HUD to your avatar
  2. Attach the meter to your avatar
  3. Make sure the HUD initializes correctly and displays a welcome message
  4. Verify that the meter displays your character information properly
  5. Test each module by accessing its functions through the HUD menu
2 Interaction testing

Test interactions between system components:

  • Test NPC recognition by approaching NPCs with different characters
  • Try interactions between different player HUDs (if possible)
  • Test accessory objects to ensure they affect the HUD stats correctly
  • Verify that the relationship system records interactions properly
3 Common troubleshooting

If you encounter issues:

  • Script errors: Check script syntax and make sure all scripts are saved properly
  • Communication issues: Verify that channel numbers match across all scripts
  • Memory errors: Make sure scripts are distributed across prims to avoid memory limits
  • UUID recognition problems: Double-check all UUID values for accuracy
  • Performance issues: Reduce script complexity or add sleep delays to intensive operations
LSL Limitations: Remember that LSL has several limitations that affect the system:
  • No break statements in loops (use boolean flags instead)
  • "event" is a reserved keyword and cannot be used as a variable
  • No ternary operators (use if-else statements)
  • Variables must be declared in the scope they're used
  • Memory limits of 64KB per script
